# Activity 2.6 - Endpoint Management (2.6.1 UEDM + 2.6.2 EDM Part 1)

## ZIG Requirement Summary

**Activity 2.6.1 - UEDM Implementation:**
- Confirm devices meet minimum compliance standards
- Asset management system for user devices (phones, desktops, laptops)
- Programmatic (API) device compliance status

**Activity 2.6.2 - EDM Part 1:**
- Migrate manual device inventory to automated EDM solution
- ZT device management from any location (with/without remote access)
- Track NPEs (Non-Person Entities) in UEM solution

## Predecessor/Successor Relationships

```
2.6.1: None → 2.6.1 (UEDM) → 2.3.6
2.6.2: None → 2.6.2 (EDM Part 1) → 2.1.2, 2.6.3, 3.4.1
```

**Note:** Activity 2.6.2 is a predecessor to Activity 2.1.2 (NPE & PKI).

## AWS Limitations

AWS Systems Manager provides endpoint management for EC2 instances but cannot:
- Manage workstations, laptops, or mobile devices (requires UEM/MDM)
- Provide cross-platform support (Windows/Mac/Linux desktops, iOS/Android)
- Integrate with enterprise IdP for device-based conditional access

These checks verify AWS SSM is configured for cloud workload management. Full UEDM/EDM compliance requires external solutions like Microsoft Intune, Jamf, or VMware Workspace ONE.

## Checks Implemented

---

### Check 2.6-A: SSM State Manager Associations

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `ssm list-associations` |
| Requirement | "Desired state configuration enforcement" |

**What it checks:** Verifies SSM State Manager has associations configured for configuration management.

**Why it matters:** State Manager ensures instances maintain desired configurations over time. Without associations, there's no automated configuration enforcement.

**Pass condition:** At least one State Manager association exists

**Fail condition:** No associations configured

**Remediation:** Create State Manager associations for configuration documents

---

### Check 2.6-B: SSM Inventory Collection

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `ssm list-associations` |
| Requirement | "Software inventory and control" |

**What it checks:** Verifies AWS-GatherSoftwareInventory association is configured.

**Why it matters:** SSM Inventory collects software, network, and system information from managed instances. This data is essential for asset management and security analysis.

**Pass condition:** AWS-GatherSoftwareInventory association exists

**Fail condition:** No inventory collection configured

**Remediation:** `aws ssm create-association --name AWS-GatherSoftwareInventory --targets Key=InstanceIds,Values=*`

---

### Check 2.6-C: SSM Default Host Management Configuration

| Field | Value |
|-------|-------|
| Severity | LOW |
| AWS API | `ssm get-service-setting` |
| Requirement | "Automatic endpoint enrollment" |

**What it checks:** Verifies Default Host Management Configuration (DHMC) is enabled.

**Why it matters:** DHMC automatically configures new EC2 instances with SSM agent and IAM role, ensuring all instances are managed from launch.

**Pass condition:** DHMC status is Customized or Default

**Fail condition:** DHMC not configured

**Remediation:** Enable DHMC in SSM settings for automatic instance management

---

### Check 2.6-D: Unmanaged EC2 Instances

| Field | Value |
|-------|-------|
| Severity | HIGH |
| Cross-reference | Activity 2.1.2-B |
| Requirement | "All endpoints managed" |

**What it checks:** Reports instances identified as unmanaged in Activity 2.1.2-B.

**Why it matters:** Unmanaged instances cannot receive endpoint policies, patches, or configuration updates. They represent security blind spots.

**Pass condition:** All EC2 instances managed by SSM (UNMANAGED_INSTANCES array empty)

**Fail condition:** Unmanaged instances exist

**Remediation:** Install SSM agent and attach IAM role (see 2.1.2-B)

**Cross-reference:** Uses `UNMANAGED_INSTANCES` and `EC2_INSTANCE_COUNT` from Activity 2.1.2

---

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| Workstation/laptop management | Requires UEM solution (Intune, Jamf) |
| Mobile device management | Requires MDM solution |
| Cross-platform support | AWS SSM is EC2-focused |
| Device-based conditional access | Requires IdP integration |
| NPE tracking for non-cloud devices | Requires external EDM |
| Remote access management | Requires VPN/ZTNA solution |

## External Tools for Full Compliance

To fully implement Activities 2.6.1 and 2.6.2, integrate AWS with:
- **UEM/MDM** - Microsoft Intune, Jamf, VMware Workspace ONE for endpoints
- **IdP** - For device-based conditional access policies
- **VPN/ZTNA** - For secure remote access regardless of location
- **CMDB** - For centralized device inventory across all platforms

## How AWS Complements External UEM

| AWS Service | Complements |
|-------------|-------------|
| SSM State Manager | UEM policy enforcement for cloud workloads |
| SSM Inventory | CMDB integration for EC2/cloud assets |
| AWS Config | Compliance reporting to enterprise dashboards |
| CloudWatch | Metrics/logs to SIEM for unified monitoring |
