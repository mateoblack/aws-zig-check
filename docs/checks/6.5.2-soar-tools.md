# Activity 6.5.2 - Implement SOAR Tools

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 6.5.2 requires:
- Develop requirements for SOAR tools
- Procure and implement SOAR solution
- Develop Implementation Plan (Integration Points, IR, Architecture, Interoperability, Scalability)
- Complete full implementation of SOAR

## Predecessor/Successor Relationships

```
6.6.2, 6.7.1 → 6.5.2 (Implement SOAR Tools) → None
```

**Predecessor:** Activity 6.6.2 - Standardized API Calls, Activity 6.7.1 - Workflow Enrichment Part 1
**Successor:** None

## AWS Limitations

AWS provides SOAR-like capabilities through multiple services:
- Security Hub for centralized findings
- EventBridge for event-driven automation
- Step Functions for workflow orchestration
- Lambda for automated response

Full SOAR typically requires:
- Dedicated SOAR platform (Splunk SOAR, Palo Alto XSOAR)
- Playbook management
- Case management
- Analyst workflow tools

## Checks Implemented

---

### Check 6.5-A: Security Hub Enabled

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `securityhub describe-hub` |
| Requirement | "Centralize security findings" |

**What it checks:** Verifies AWS Security Hub is enabled.

**Why it matters:** Security Hub aggregates findings from multiple AWS services and third-party tools, serving as the foundation for SOAR integration.

**Pass condition:** Security Hub is enabled

**Fail condition:** Security Hub not enabled

**Remediation:** Enable Security Hub in the Security Hub console

---

### Check 6.5-B: Security Hub Standards Enabled

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `securityhub get-enabled-standards` |
| Requirement | "Automated policy enforcement" |

**What it checks:** Verifies security standards are enabled (AWS Foundational, CIS, etc.).

**Why it matters:** Standards provide automated security checks that feed into SOAR workflows for remediation.

**Pass condition:** At least one standard enabled

**Fail condition:** No standards enabled

**Remediation:** Enable AWS Foundational Security Best Practices or CIS AWS Foundations Benchmark

---

### Check 6.5-C: Security Hub Automation Rules

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `securityhub list-automation-rules` |
| Requirement | "Automated response and remediation" |

**What it checks:** Counts Security Hub automation rules for automated finding handling.

**Why it matters:** Automation rules enable automatic suppression, severity updates, or workflow triggers based on finding attributes.

**Pass condition:** Automation rules configured

**Note:** Optional - automation can also be done via EventBridge

**Remediation:** Create automation rules for common finding patterns

---

### Check 6.5-D: EventBridge Security Rules

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `events list-rules` |
| Requirement | "Triggering playbooks for automated response" |

**What it checks:** Identifies EventBridge rules for security events.

**Why it matters:** EventBridge enables event-driven automation, triggering Lambda functions or Step Functions for incident response.

**Pass condition:** Security-related EventBridge rules exist

**Note:** Informational - checks for rules with security-related names

**Remediation:** Create EventBridge rules to trigger on Security Hub findings or GuardDuty events

---

### Check 6.5-E: Step Functions for Orchestration

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `stepfunctions list-state-machines` |
| Requirement | "Workflow orchestration" |

**What it checks:** Counts Step Functions state machines available for orchestration.

**Why it matters:** Step Functions enable complex multi-step workflows for incident response, similar to SOAR playbooks.

**Pass condition:** State machines exist

**Note:** Optional - simple automation can use Lambda directly

**Remediation:** Create Step Functions for complex IR workflows

---

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| SOAR playbook management | Requires dedicated SOAR platform |
| Case management | Requires external tooling |
| Analyst workflow | Requires SOAR UI |
| Integration with all security tools | Environment-specific |

## External Tools for Full Compliance

To fully implement Activity 6.5.2:
- **SOAR Platforms** - Splunk SOAR, Palo Alto XSOAR, Swimlane
- **Ticketing** - ServiceNow, Jira for case management
- **Runbooks** - AWS Systems Manager Automation Documents

## How AWS Provides SOAR Capabilities

| ZIG Concept | AWS Equivalent |
|-------------|----------------|
| Centralized Findings | Security Hub |
| Automated Response | EventBridge + Lambda |
| Workflow Orchestration | Step Functions |
| Playbooks | Systems Manager Automation |
| Threat Intelligence | GuardDuty |
