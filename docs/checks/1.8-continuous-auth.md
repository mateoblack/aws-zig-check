# Activity 1.8 - Continuous Authentication

## ZIG Requirement Summary

Per NSA ZIG Phase One, Capability 1.8 requires:
- Standardizing legacy single authentication to an organizationally approved IdP
- Rule-based (time) authentication with periodic re-authentication
- Continuous authentication based on application/software activities and privileges requested
- Real-time anomaly detection during sessions
- Automatic session termination on authentication failure or suspicious behavior

## Important Note: External Tooling Required

**Full continuous authentication requires external tooling.** AWS does not provide native:
- Real-time behavioral biometrics
- Session hijacking detection
- Risk-based step-up authentication
- User and Entity Behavior Analytics (UEBA)

These require external solutions like:
- Identity Providers: Okta Adaptive MFA, Azure AD Conditional Access, Ping Identity
- UEBA/SIEM: Splunk UBA, Microsoft Sentinel, Exabeam
- Session Management: CyberArk, BeyondTrust

**The checks below verify AWS is configured to support continuous authentication principles**, but do not validate a complete implementation.

## Checks Implemented

### Check 1.8.1-A: IAM Role Maximum Session Duration

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `iam list-roles`, `iam get-role` |
| Fields Checked | `MaxSessionDuration` |

**What it checks:** Identifies IAM roles with maximum session duration greater than 4 hours (14400 seconds).

**Why it matters:** ZIG requires "periodic re-authentication at fixed intervals based on time and session duration." Long session durations:
- Allow extended access without re-verification
- Increase window for session hijacking
- Reduce opportunities to detect compromised credentials

Shorter session durations force more frequent credential refresh, aligning with continuous authentication principles.

**Pass condition:** All IAM roles have `MaxSessionDuration` ≤ 4 hours (14400 seconds)

**Fail condition:** One or more roles allow sessions longer than 4 hours

**Note:** AWS default is 1 hour, maximum is 12 hours. Roles requiring longer sessions should be documented with justification.

**Remediation:**
```bash
# List roles with long session durations
for role in $(aws iam list-roles --query 'Roles[?MaxSessionDuration > `14400`].RoleName' --output text); do
  echo "$role: $(aws iam get-role --role-name $role --query 'Role.MaxSessionDuration')"
done

# Update role session duration
aws iam update-role --role-name <role-name> --max-session-duration 14400
```

### Check 1.8.1-B: GuardDuty Enabled for Anomaly Detection

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `guardduty list-detectors`, `guardduty get-detector` |
| Fields Checked | Detector existence and status |

**What it checks:** Whether Amazon GuardDuty is enabled for threat detection.

**Why it matters:** ZIG requires "real-time anomaly detection" and "continuous authentication policies that monitor the session for real-time anomalies." GuardDuty provides:
- Credential compromise detection (unusual API calls, locations)
- Unauthorized access attempts
- Instance credential exfiltration detection
- Anomalous behavior patterns

GuardDuty finding types relevant to continuous auth:
- `UnauthorizedAccess:IAMUser/InstanceCredentialExfiltration`
- `UnauthorizedAccess:IAMUser/ConsoleLoginSuccess.B`
- `UnauthorizedAccess:IAMUser/MaliciousIPCaller`

**Pass condition:** GuardDuty detector exists and is enabled

**Fail condition:** No GuardDuty detector or detector is disabled

**Remediation:**
```bash
# Enable GuardDuty
aws guardduty create-detector --enable

# Verify status
aws guardduty list-detectors
aws guardduty get-detector --detector-id <detector-id>
```

### Check 1.8.1-C: CloudTrail Multi-Region for Session Monitoring

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `cloudtrail describe-trails`, `cloudtrail get-trail-status` |
| Fields Checked | `IsMultiRegionTrail`, `IsLogging` |

**What it checks:** Whether CloudTrail is configured with multi-region logging enabled.

**Why it matters:** ZIG requires monitoring "application and software activity" during sessions. CloudTrail provides:
- Complete audit trail of API calls
- Session activity logging (AssumeRole, console logins)
- Data for anomaly detection and forensics
- Integration with GuardDuty and Security Hub

Multi-region trails ensure no activity is missed regardless of where it occurs.

**Pass condition:** At least one multi-region trail exists and is actively logging

**Fail condition:** No multi-region trail or trail is not logging

**Remediation:**
```bash
# Create multi-region trail
aws cloudtrail create-trail \
  --name organization-trail \
  --s3-bucket-name <bucket-name> \
  --is-multi-region-trail \
  --enable-log-file-validation

# Start logging
aws cloudtrail start-logging --name organization-trail
```

### Check 1.8.1-D: IAM Policies with MFA Age Conditions

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `iam list-policies`, `iam get-policy-version` |
| Fields Checked | Policy conditions for `aws:MultiFactorAuthAge` |

**What it checks:** Whether IAM policies enforce re-authentication based on MFA age.

**Why it matters:** ZIG requires "rule-based (time) authentication" with periodic re-verification. The `aws:MultiFactorAuthAge` condition:
- Limits how long MFA authentication remains valid
- Forces re-authentication after specified seconds
- Implements time-based continuous authentication at the policy level

Example: `"aws:MultiFactorAuthAge": "3600"` requires MFA re-auth every hour.

**Pass condition:** At least one policy uses `aws:MultiFactorAuthAge` condition

**Fail condition:** No policies enforce MFA age limits

**Note:** This is a MEDIUM finding because MFA age enforcement may be handled at the IdP level (Okta, Azure AD) rather than in AWS policies.

**Remediation:**
```bash
# Add MFA age condition to policy
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "*",
      "Resource": "*",
      "Condition": {
        "NumericGreaterThan": {
          "aws:MultiFactorAuthAge": "3600"
        }
      }
    }
  ]
}
```

### Check 1.8.1-E: Security Hub Enabled

| Field | Value |
|-------|-------|
| Severity | LOW |
| AWS API | `securityhub describe-hub` |
| Fields Checked | Hub existence and status |

**What it checks:** Whether AWS Security Hub is enabled for centralized security findings.

**Why it matters:** ZIG requires "security analysts review incidents" and continuous monitoring. Security Hub:
- Aggregates findings from GuardDuty, IAM Access Analyzer, and other services
- Provides unified view of security posture
- Enables automated response workflows
- Supports compliance standards (CIS, PCI-DSS)

**Pass condition:** Security Hub is enabled

**Fail condition:** Security Hub is not enabled

**Remediation:**
```bash
# Enable Security Hub
aws securityhub enable-security-hub

# Enable specific standards
aws securityhub batch-enable-standards \
  --standards-subscription-requests '[{"StandardsArn": "arn:aws:securityhub:::ruleset/cis-aws-foundations-benchmark/v/1.2.0"}]'
```

### Check 1.8.1-F: IAM Identity Center Session Duration

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `sso-admin list-instances`, `sso-admin describe-instance-access-control-attribute-configuration` |
| Fields Checked | SSO session settings |

**What it checks:** Whether IAM Identity Center (SSO) is configured with appropriate session duration limits.

**Why it matters:** ZIG requires "periodic re-authentication at fixed intervals." IAM Identity Center session settings control:
- How long users remain authenticated
- When re-authentication is required
- Session timeout behavior

**Pass condition:** IAM Identity Center is configured (session settings managed via SSO)

**Fail condition:** IAM Identity Center not configured (no centralized session management)

**Note:** Specific session duration settings are configured in the IAM Identity Center console or via external IdP. This check verifies SSO is in use for centralized session management.

**Remediation:**
```bash
# Enable IAM Identity Center via Console
# 1. Go to IAM Identity Center
# 2. Settings → Session settings
# 3. Configure session duration (recommend ≤ 8 hours)
# 4. Configure MFA requirements
```

### Check 1.8.1-G: GuardDuty Credential Findings

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `guardduty list-findings` |
| Fields Checked | UnauthorizedAccess and CredentialAccess finding types |

**What it checks:** Whether there are active GuardDuty findings related to credential compromise or unauthorized access.

**Why it matters:** ZIG describes a scenario where "the system detects an unusual change in User/PE behavior" and "the session is immediately terminated." Active credential-related findings indicate:
- Potential session hijacking attempts
- Credential compromise
- Anomalous authentication patterns

These findings require immediate investigation and response.

**Pass condition:** No active HIGH/CRITICAL credential-related GuardDuty findings

**Fail condition:** Active findings exist for credential compromise or unauthorized access

**Remediation:**
```bash
# List credential-related findings
aws guardduty list-findings --detector-id <detector-id> \
  --finding-criteria '{"Criterion": {"type": {"Eq": ["UnauthorizedAccess:IAMUser/ConsoleLoginSuccess.B", "UnauthorizedAccess:IAMUser/InstanceCredentialExfiltration.OutsideAWS"]}}}'

# Get finding details
aws guardduty get-findings --detector-id <detector-id> --finding-ids <finding-id>

# Investigate and remediate:
# 1. Identify compromised credentials
# 2. Rotate/revoke affected credentials
# 3. Review CloudTrail for unauthorized actions
# 4. Archive finding after remediation
```

## Checks NOT Implemented

| Continuous Auth Feature | Reason |
|------------------------|--------|
| Real-time behavioral biometrics | External IdP capability (Okta, Azure AD) |

| User behavior analytics | External UEBA (Splunk UBA, Exabeam) |
| Biometric re-authentication prompts | External IdP/device capability |
| Real-time privilege evaluation | External PAM/PIM tool |
