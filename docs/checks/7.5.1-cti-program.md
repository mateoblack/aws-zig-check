# Activity 7.5.1 - Cyber Threat Intelligence (CTI) Program Part 1

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 7.5.1 requires:
- Develop CTI program policy, standard, and process
- Establish CTI teams with key stakeholders
- Gather intelligence from common threat feeds into SIEM
- Build integration points with device and network PEP/PDP

## Predecessor/Successor Relationships

```
None → 7.5.1 (CTI Program Part 1) → 7.2.2, 7.5.2
```

**Predecessor:** None
**Successor:** Activity 7.2.2 - Threat Alerting Part 2, Activity 7.5.2 - CTI Program Part 2

## AWS Limitations

AWS provides threat intelligence through:
- GuardDuty built-in threat feeds
- GuardDuty custom threat lists
- Security Hub third-party integrations

Full CTI program requires:
- Documented CTI policy
- CTI team with stakeholders
- External threat feed integration

## Checks Implemented

---

### Check 7.5-A: GuardDuty Threat Lists

| Field | Value |
|-------|-------|
| Severity | HIGH (if GuardDuty disabled) / INFO |
| AWS API | `guardduty list-threat-intel-sets`, `guardduty list-ip-sets` |
| Requirement | "CTI feeds integrated into SIEM" |

**What it checks:** Verifies GuardDuty has custom threat intelligence lists configured.

**Why it matters:** Custom threat lists allow you to add organization-specific indicators (malicious IPs, domains) beyond GuardDuty's built-in intelligence.

**Pass condition:** Custom threat lists or IP sets configured

**Note:** GuardDuty includes built-in threat intel even without custom lists

**Remediation:** Upload custom threat lists for your environment

---

### Check 7.5-B: Security Hub Third-Party Integrations

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `securityhub list-enabled-products-for-import` |
| Requirement | "Integration points across pillars" |

**What it checks:** Verifies third-party security products are integrated with Security Hub.

**Why it matters:** Third-party integrations bring additional threat intelligence and findings into the centralized Security Hub.

**Pass condition:** Product integrations enabled

**Note:** Informational - AWS native services integrate automatically

**Remediation:** Enable integrations from Security Hub console

---

### Check 7.5-C: GuardDuty Malware Protection

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `guardduty get-detector` |
| Requirement | "Threat detection capabilities" |

**What it checks:** Verifies GuardDuty Malware Protection is enabled for EBS scanning.

**Why it matters:** Malware Protection scans EBS volumes for malware when GuardDuty detects suspicious behavior.

**Pass condition:** Malware Protection enabled

**Note:** Optional - additional cost

**Remediation:** Enable Malware Protection in GuardDuty settings

---

### Check 7.5-D: GuardDuty S3 Protection

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `guardduty get-detector` |
| Requirement | "Data protection threat detection" |

**What it checks:** Verifies GuardDuty S3 Protection is enabled.

**Why it matters:** S3 Protection monitors S3 data events for suspicious access patterns and potential data exfiltration.

**Pass condition:** S3 Protection enabled

**Note:** Recommended for environments with sensitive S3 data

**Remediation:** Enable S3 Protection in GuardDuty settings

## External Tools for Full Compliance

To fully implement Activity 7.5.1:
- **Threat Intelligence Platforms** - MISP, ThreatConnect, Recorded Future
- **Threat Feeds** - AlienVault OTX, Abuse.ch, CISA AIS
- **STIX/TAXII** - For standardized threat intel sharing

## How AWS Provides CTI

| ZIG Concept | AWS Equivalent |
|-------------|----------------|
| Threat Intelligence | GuardDuty built-in feeds |
| Custom Threat Lists | GuardDuty threat intel sets |
| CTI Integration | Security Hub integrations |
| Malware Detection | GuardDuty Malware Protection |

## GuardDuty Threat Intelligence Sources

GuardDuty uses multiple threat intelligence sources:
- AWS internal threat intelligence
- CrowdStrike
- Proofpoint
- Custom threat lists (user-provided)

## Recommended Custom Threat Lists

Consider adding these to GuardDuty:
- Known malicious IPs from your industry ISAC
- Indicators from recent incidents
- Tor exit nodes (if Tor is not expected)
- Known C2 infrastructure
