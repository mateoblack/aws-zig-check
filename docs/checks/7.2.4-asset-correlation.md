# Activity 7.2.4 - Asset ID and Alert Correlation

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 7.2.4 requires:
- Identify all assets in SIEM with relevant metadata
- Correlate assets with threat alerts
- Enable ZT-driven alert triage and asset containment
- Optimize IR decision-making with asset-aware correlation

## Predecessor/Successor Relationships

```
7.1.2 (Log Parsing) → 7.2.4 (Asset ID and Alert Correlation) → None
```

**Predecessor:** Activity 7.1.2 - Log Parsing
**Successor:** None

## AWS Limitations

AWS provides asset inventory through:
- AWS Config for resource inventory and compliance
- Systems Manager for managed instance inventory
- Resource tagging for asset identification

Full ZIG compliance requires:
- CMDB integration
- Asset criticality classification
- Real-time asset correlation in SIEM

## Checks Implemented

---

### Check 7.2.4-A: AWS Config Enabled

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `configservice describe-configuration-recorders` |
| Requirement | "Comprehensive asset inventory" |

**What it checks:** Verifies AWS Config is enabled and recording resource configurations.

**Why it matters:** AWS Config provides continuous inventory of AWS resources, configuration history, and compliance monitoring.

**Pass condition:** Config recorder enabled and recording

**Fail condition:** Config not enabled or not recording

**Remediation:** Enable AWS Config in the Config console

---

### Check 7.2.4-B: AWS Config Rules

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `configservice describe-config-rules` |
| Requirement | "Asset compliance monitoring" |

**What it checks:** Verifies AWS Config rules are configured for compliance monitoring.

**Why it matters:** Config rules evaluate resource configurations against desired settings, identifying non-compliant assets.

**Pass condition:** Config rules exist

**Note:** Informational - rules should align with security requirements

**Remediation:** Enable managed rules or create custom rules

---

### Check 7.2.4-C: Systems Manager Inventory

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `ssm describe-instance-information` |
| Requirement | "Device inventory" |

**What it checks:** Verifies Systems Manager is managing instances for inventory collection.

**Why it matters:** SSM provides detailed inventory of installed software, patches, and configurations on managed instances.

**Pass condition:** Instances managed by SSM

**Note:** Informational - requires SSM agent on instances

**Remediation:** Install SSM agent and configure inventory collection

---

### Check 7.2.4-D: Resource Tagging Coverage

| Field | Value |
|-------|-------|
| Severity | LOW |
| AWS API | `ec2 describe-instances` |
| Requirement | "Asset identification" |

**What it checks:** Verifies EC2 instances have Name tags for identification.

**Why it matters:** Tags enable asset identification in logs and alerts, supporting faster incident response.

**Pass condition:** ≥80% of instances have Name tag

**Fail condition:** <80% tagging coverage

**Remediation:** Implement tagging policy and tag existing resources

---

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| CMDB integration | Requires external tooling |
| Asset criticality classification | Requires manual assessment |
| Real-time SIEM correlation | Environment-specific |

## External Tools for Full Compliance

To fully implement Activity 7.2.4:
- **CMDB** - ServiceNow, BMC for asset management
- **SIEM** - Splunk, Elastic for correlation
- **Asset Discovery** - Qualys, Tenable for network scanning

## How AWS Provides Asset Correlation

| ZIG Concept | AWS Equivalent |
|-------------|----------------|
| Asset Inventory | AWS Config, SSM Inventory |
| Asset Identification | Resource Tags, ARNs |
| Compliance Monitoring | Config Rules |
| Alert Correlation | Security Hub, CloudWatch |

## Recommended Tags for Asset Identification

| Tag Key | Purpose |
|---------|---------|
| Name | Human-readable identifier |
| Environment | prod, staging, dev |
| Owner | Team or individual responsible |
| CostCenter | Financial tracking |
| DataClassification | Sensitivity level |
| Compliance | Regulatory requirements |
