# Activity 7.3.1 - Implement Analytics Tools

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 7.3.1 requires:
- Define requirements for analytic tool capabilities
- Evaluate and select analytic tools
- Implement and integrate analytics tools
- Continuously improve analytics capabilities

## Predecessor/Successor Relationships

```
7.1.2 (Log Parsing) → 7.3.1 (Implement Analytics Tools) → None
```

**Predecessor:** Activity 7.1.2 - Log Parsing
**Successor:** None

## AWS Limitations

AWS provides analytics through multiple services:
- Amazon Detective for security investigation
- CloudWatch Anomaly Detection for ML-based alerting
- CloudWatch Logs Insights for log analysis
- Athena for SQL-based log queries

Full UEBA/analytics typically requires:
- Dedicated analytics platform
- ML model training
- Behavioral baseline development

## Checks Implemented

---

### Check 7.3-A: Amazon Detective

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `detective list-graphs` |
| Requirement | "Analytics tools for threat detection" |

**What it checks:** Verifies Amazon Detective is enabled for security investigation.

**Why it matters:** Detective automatically analyzes and visualizes security data to help investigate potential security issues.

**Pass condition:** Detective behavior graph exists

**Note:** Optional - requires GuardDuty

**Remediation:** Enable Detective from the Detective console

---

### Check 7.3-B: CloudWatch Anomaly Detection

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `cloudwatch describe-anomaly-detectors` |
| Requirement | "ML-based threat detection" |

**What it checks:** Verifies CloudWatch anomaly detectors are configured.

**Why it matters:** Anomaly detection uses ML to identify unusual metric patterns without manual threshold configuration.

**Pass condition:** Anomaly detectors exist

**Note:** Optional - useful for dynamic workloads

**Remediation:** Create anomaly detection alarms in CloudWatch

---

### Check 7.3-C: CloudWatch Logs Insights Queries

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `logs describe-query-definitions` |
| Requirement | "Log analysis capabilities" |

**What it checks:** Verifies saved Logs Insights queries exist.

**Why it matters:** Saved queries enable consistent, repeatable log analysis for security investigations.

**Pass condition:** Saved queries exist

**Note:** Informational - queries can be run ad-hoc

**Remediation:** Save common security analysis queries

---

### Check 7.3-D: Athena Workgroups

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `athena list-work-groups` |
| Requirement | "Big data analytics" |

**What it checks:** Verifies Athena workgroups are configured beyond default.

**Why it matters:** Athena enables SQL queries against CloudTrail, VPC Flow Logs, and other log data in S3.

**Pass condition:** Custom workgroups exist

**Note:** Informational - default workgroup is always present

**Remediation:** Create workgroups for security analysis

## Useful Logs Insights Queries

Save these queries for security analysis:

**Top 10 Error Codes:**
```
fields @timestamp, @message
| filter @message like /error/i
| stats count(*) as count by errorCode
| sort count desc
| limit 10
```

**Failed Console Logins:**
```
fields @timestamp, userIdentity.userName, sourceIPAddress
| filter eventName = "ConsoleLogin" and errorMessage = "Failed authentication"
| sort @timestamp desc
```

**IAM Policy Changes:**
```
fields @timestamp, userIdentity.userName, eventName, requestParameters.policyName
| filter eventName like /Policy/
| sort @timestamp desc
```
