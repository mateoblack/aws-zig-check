# Activity 6.7.1 - Workflow Enrichment Part 1

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 6.7.1 requires:
- Establish cybersecurity IR guidance using NIST best practices
- Identify threat events utilizing Enterprise guidance
- Enable workflows for security events using internal context and threat intelligence
- Identify approved external sources of enrichment for future integration

## Predecessor/Successor Relationships

```
None → 6.7.1 (Workflow Enrichment Part 1) → 6.5.2, 6.7.2
```

**Predecessor:** None (Consider completing Activity 7.5.1 - CTI Program Part 1)
**Successor:** Activity 6.5.2 - Implement SOAR Tools, Activity 6.7.2 - Workflow Enrichment Part 2

## AWS Limitations

AWS provides threat intelligence and workflow capabilities:
- GuardDuty for threat detection and CTI
- Security Hub for findings aggregation
- EventBridge for event-driven workflows
- CloudWatch for alerting

Full ZIG compliance requires:
- Documented IR procedures
- CTI policy
- Integration with external threat feeds

## Checks Implemented

---

### Check 6.7-A: GuardDuty Enabled

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `guardduty list-detectors`, `guardduty get-detector` |
| Requirement | "Threat intelligence enrichment" |

**What it checks:** Verifies GuardDuty is enabled for continuous threat detection.

**Why it matters:** GuardDuty provides threat intelligence, behavioral analysis, and anomaly detection - essential for workflow enrichment.

**Pass condition:** GuardDuty detector enabled

**Fail condition:** GuardDuty not enabled or disabled

**Remediation:** Enable GuardDuty in the GuardDuty console

---

### Check 6.7-B: Security Hub Findings Aggregation

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `securityhub describe-hub`, `securityhub list-enabled-products-for-import` |
| Requirement | "Centralized threat event identification" |

**What it checks:** Verifies Security Hub is enabled and aggregating findings from multiple sources.

**Why it matters:** Security Hub centralizes findings from GuardDuty, Inspector, Macie, and third-party tools for unified workflow processing.

**Pass condition:** Security Hub enabled with product integrations

**Fail condition:** Security Hub not enabled

**Remediation:** Enable Security Hub and configure product integrations

---

### Check 6.7-C: Security CloudWatch Alarms

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `cloudwatch describe-alarms` |
| Requirement | "Security event workflows" |

**What it checks:** Identifies CloudWatch alarms for security-related events.

**Why it matters:** Alarms trigger notifications and automated responses for security events like unauthorized API calls or root login.

**Pass condition:** Security-related alarms configured

**Fail condition:** No security alarms detected

**Remediation:** Create alarms for CloudTrail metrics (unauthorized calls, root login, etc.)

---

### Check 6.7-D: Security SNS Topics

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `sns list-topics` |
| Requirement | "IR notification workflows" |

**What it checks:** Identifies SNS topics for security notifications.

**Why it matters:** SNS topics enable notification workflows to security teams, ticketing systems, or automated response functions.

**Pass condition:** Security-related SNS topics exist

**Note:** Informational - notifications may use other mechanisms

**Remediation:** Create SNS topics for security alerting and subscribe appropriate endpoints

---

### Check 6.7-E: Security Lambda Functions

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `lambda list-functions` |
| Requirement | "Automated IR workflows" |

**What it checks:** Identifies Lambda functions for security automation.

**Why it matters:** Lambda functions enable automated response actions like isolating instances, revoking credentials, or enriching findings.

**Pass condition:** Security-related Lambda functions exist

**Note:** Informational - automation may use other mechanisms

**Remediation:** Create Lambda functions for automated incident response

## Recommended CloudWatch Alarms

For ZIG-compliant security monitoring:

| Alarm | CloudTrail Metric Filter |
|-------|-------------------------|
| Root Login | `{ $.userIdentity.type = "Root" }` |
| Unauthorized API | `{ $.errorCode = "*UnauthorizedAccess*" }` |
| Console Login Failures | `{ $.eventName = "ConsoleLogin" && $.errorMessage = "Failed*" }` |
| IAM Policy Changes | `{ $.eventName = "Put*Policy" || $.eventName = "Delete*Policy" }` |
| Security Group Changes | `{ $.eventName = "AuthorizeSecurityGroup*" }` |
