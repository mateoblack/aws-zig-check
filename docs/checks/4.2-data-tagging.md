# Activity 4.2.1/4.3.1 - Data Tagging & Classification

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activities 4.2.1 and 4.3.1 require:
- Define data tagging standards and control vocabulary
- Implement data tagging and classification tools
- Establish machine-readable tags aligned with Enterprise standards
- Enable automated data discovery and classification

## Predecessor/Successor Relationships

```
None → 4.2.1 (Data Tagging Standards) → 4.3.1 (Classification Tools) → 4.6.1 (Enforcement Points)
```

**Predecessor (4.2.1):** None - foundational activity
**Successor (4.2.1):** Activity 4.3.1 - Implement Data Tagging Tools
**Predecessor (4.3.1):** Activity 4.2.1 - Define Data Tagging Standards
**Successor (4.3.1):** Activity 4.6.1 - Implement Enforcement Points

## Compliance Tag Convention

This checker uses **tag-based compliance detection**. Resources with these tags are held to stricter standards:

| Tag Key | Compliance Values |
|---------|-------------------|
| `DataClassification` | `critical`, `sensitive`, `pii`, `phi`, `pci`, `high` |
| `Compliance` | `required`, `critical`, `high` |
| `SecurityLevel` | `critical`, `sensitive`, `high` |

**Example:** A bucket tagged `DataClassification=critical` will be checked for encryption, access logging, Object Lock, etc.

## AWS Limitations

AWS provides basic data discovery (Macie) but not comprehensive data governance:
- No built-in data taxonomy/vocabulary management
- Limited classification beyond PII/credential detection
- No data lineage tracking
- Requires external tools for full data governance

## Checks Implemented

---

### Check 4.2-A: Macie Enabled

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `macie2 get-macie-session` |
| Requirement | "Implement data tagging and classification tools" |

**What it checks:** Verifies Amazon Macie is enabled for automated data discovery.

**Why it matters:** Macie provides automated sensitive data discovery (PII, credentials, etc.) - the closest AWS-native equivalent to data classification tools.

**Pass condition:** Macie session status is ENABLED

**Fail condition:** Macie not enabled

**Remediation:** `aws macie2 enable-macie`

---

### Check 4.2-B: Macie Sensitive Data Findings

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `macie2 list-findings` |
| Requirement | "Identify and classify sensitive data" |

**What it checks:** Queries Macie for classification findings (PII, credentials detected).

**Why it matters:** Sensitive data findings indicate unprotected PII or credentials that need remediation.

**Pass condition:** No Macie classification findings

**Fail condition:** One or more sensitive data findings exist

**Remediation:** Review findings and remediate (encrypt, restrict access, or remove sensitive data)

---

### Check 4.2-C: Glue Data Catalog

| Field | Value |
|-------|-------|
| Severity | LOW |
| AWS API | `glue get-databases`, `glue search-tables` |
| Requirement | "Establish data inventory" |

**What it checks:** Verifies Glue Data Catalog is configured for centralized data inventory.

**Why it matters:** Data Catalog provides metadata management and can store classification/tagging information.

**Pass condition:** At least one Glue database exists

**Fail condition:** No Glue Data Catalog databases

**Remediation:** Configure Glue Data Catalog for data inventory

---

### Check 4.2-D: S3 Bucket Tagging Coverage

| Field | Value |
|-------|-------|
| Severity | LOW |
| AWS API | `s3api list-buckets`, `s3api get-bucket-tagging` |
| Requirement | "Implement data tagging standards" |

**What it checks:** Identifies S3 buckets without any tags.

**Why it matters:** Untagged buckets cannot be classified or have compliance policies applied. Per ZIG 4.2.1, all data should be tagged.

**Pass condition:** All S3 buckets have at least one tag

**Fail condition:** Buckets exist without any tags

**Remediation:** Tag buckets with `DataClassification`, `Compliance`, or other relevant tags

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| Data taxonomy management | Organizational process, not API-queryable |
| Control vocabulary | Requires external data governance tools |
| ML-based classification | Macie is limited; full ML needs external tools |
| Data lineage | Requires data governance platforms |
| Tag validation rules | Requires custom implementation |

## External Tools for Full Compliance

To fully implement Activities 4.2.1/4.3.1, integrate AWS with:
- **Data Governance** - Collibra, Alation, AWS DataZone for taxonomy/vocabulary
- **Data Classification** - Microsoft Purview, BigID for advanced classification
- **Data Catalog** - AWS Glue + external enrichment
- **Tag Management** - AWS Tag Editor, custom tag policies

## How AWS Complements External Data Governance

| AWS Service | Complements |
|-------------|-------------|
| Macie | Automated PII/credential discovery → classification input |
| Glue Data Catalog | Metadata repository → data governance integration |
| Resource Tags | Classification labels → policy enforcement |
| Lake Formation | Fine-grained access → tag-based permissions |
