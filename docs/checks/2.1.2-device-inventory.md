# Activity 2.1.2 - Device Inventory & NPE Management

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 2.1.2 requires:
- Maintain a comprehensive inventory of all devices (CMDB)
- Track Non-Person Entities (NPEs) including service accounts and IoT devices
- Implement PKI/X.509 certificate management for device authentication
- Ensure all devices are managed and can receive security policies

## Predecessor/Successor Relationships

```
2.6.2 (EDM Part 1) → 2.1.2 (NPE & PKI) → 2.4.1 (Deny Device by Default)
```

**Predecessor:** Activity 2.6.2 - Enterprise Device Management should be in place first.
**Successor:** Activity 2.4.1 - Deny Device by Default depends on device inventory.

## AWS Limitations

AWS APIs can only inventory cloud-native resources. On-premises devices, workstations, and mobile devices require external CMDB/UEM solutions. These checks focus on AWS-managed infrastructure.

## Checks Implemented

---

### Check 2.1.2-A: AWS Config Enabled (CMDB Equivalent)

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `configservice describe-configuration-recorders`, `configservice describe-configuration-recorder-status` |
| Requirement | "Maintain comprehensive device inventory" |

**What it checks:** Verifies AWS Config is enabled and actively recording resource configurations.

**Why it matters:** AWS Config serves as the cloud CMDB, tracking all AWS resources and their configurations over time. Without Config, there's no centralized inventory of cloud resources.

**Pass condition:** AWS Config recorder exists AND is actively recording

**Fail condition:** No Config recorder OR recorder is not recording

**Remediation:** Enable AWS Config with a delivery channel to S3/SNS

---

### Check 2.1.2-B: SSM Managed Instance Coverage

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `ec2 describe-instances`, `ssm describe-instance-information` |
| Requirement | "All devices managed and can receive security policies" |

**What it checks:** Compares running EC2 instances against SSM-managed instances to identify unmanaged servers.

**Why it matters:** Unmanaged instances cannot receive patches, configuration policies, or be inventoried through SSM. They represent blind spots in device management.

**Pass condition:** All running EC2 instances appear in SSM managed instance list

**Fail condition:** One or more EC2 instances not managed by SSM

**Remediation:** Install SSM agent and attach IAM role with `AmazonSSMManagedInstanceCore` policy

**Cross-reference:** Results stored in `UNMANAGED_INSTANCES` array for use by Activity 2.6 checks

---

### Check 2.1.2-C: ACM Private CA (Enterprise PKI)

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `acm-pca list-certificate-authorities` |
| Requirement | "PKI/X.509 certificate management for device authentication" |

**What it checks:** Verifies ACM Private CA is configured for enterprise certificate issuance.

**Why it matters:** Zero Trust requires strong device identity. ACM Private CA enables X.509 certificate issuance for mutual TLS, device authentication, and code signing.

**Pass condition:** At least one ACTIVE ACM Private CA exists

**Fail condition:** No ACM Private CA configured

**Remediation:** Create ACM Private CA for enterprise PKI needs (or document external PKI)

---

### Check 2.1.2-D: ACM Certificate Inventory

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `acm list-certificates`, `acm describe-certificate` |
| Requirement | "Track certificate lifecycle" |

**What it checks:** Inventories ACM certificates and identifies any expiring within 30 days.

**Why it matters:** Expired certificates cause service outages and security gaps. Proactive monitoring prevents certificate-related incidents.

**Pass condition:** All ACM certificates have >30 days until expiration

**Fail condition:** One or more certificates expiring within 30 days

**Remediation:** Renew expiring certificates or enable ACM auto-renewal for eligible certificates

---

### Check 2.1.2-E: IoT Device Registry

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `iot list-things`, `iot list-thing-principals` |
| Requirement | "Track Non-Person Entities (NPEs)" |

**What it checks:** If IoT Core is in use, verifies all IoT things have X.509 certificates attached.

**Why it matters:** IoT devices are NPEs that require strong identity. Devices without certificates cannot be authenticated or authorized properly.

**Pass condition:** All IoT things have at least one certificate principal attached

**Fail condition:** IoT things exist without certificate attachments

**Remediation:** Attach X.509 certificates to IoT devices for authentication

## Global Variables Set

These variables are set by Activity 2.1.2 checks for use by subsequent device pillar checks:

| Variable | Description |
|----------|-------------|
| `EC2_INSTANCE_COUNT` | Total running EC2 instances |
| `SSM_MANAGED_COUNT` | Instances managed by SSM |
| `UNMANAGED_INSTANCES` | Array of instance IDs not in SSM |
| `CONFIG_RECORDER_ENABLED` | Boolean for AWS Config status |

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| On-premises device inventory | Requires external CMDB (ServiceNow, etc.) |
| Workstation/laptop tracking | Requires UEM solution (Intune, Jamf, etc.) |
| Mobile device management | Requires MDM solution |
| Hardware asset tracking | Physical inventory - not API queryable |
| Device health attestation | Requires endpoint agent integration |

## External Tools for Full Compliance

To fully implement Activity 2.1.2, integrate AWS with:
- **CMDB** - ServiceNow, BMC Helix for enterprise asset inventory
- **UEM/MDM** - For workstation, laptop, and mobile device tracking
- **Enterprise PKI** - For X.509 certificate lifecycle management (ACM-PCA handles cloud)
- **NAC** - For device registration and network access control

## How AWS Complements External CMDB

| AWS Service | Complements |
|-------------|-------------|
| AWS Config | Cloud resource inventory → CMDB sync |
| SSM Inventory | EC2 software/config → CMDB enrichment |
| ACM/ACM-PCA | Certificate inventory → PKI integration |
| IoT Core | IoT device registry → CMDB for NPEs |
