# Activity 6.6.2 - Standardized API Calls and Schemas Part 1

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 6.6.2 requires:
- Establish API standard with approved patterns and protocols
- Identify existing APIs and update to standard
- Implement automated pattern and protocol services
- Support OAuth 2.0, OpenID Connect for authentication

## Predecessor/Successor Relationships

```
None → 6.6.2 (Standardized API Calls Part 1) → 5.2.2, 6.5.2, 6.6.3
```

**Predecessor:** None
**Successor:** Activity 5.2.2 - Implement SDN, Activity 6.5.2 - Implement SOAR, Activity 6.6.3 - Standardized API Part 2

## AWS Limitations

AWS API Gateway provides API management but:
- Doesn't enforce organization-wide API standards automatically
- Requires manual configuration of authorization
- WAF integration is optional

Full ZIG compliance requires:
- Documented API standards
- API governance process
- Regular API audits

## Checks Implemented

---

### Check 6.6-A: API Gateway Configured

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `apigateway get-rest-apis`, `apigatewayv2 get-apis` |
| Requirement | "Establish API standard" |

**What it checks:** Counts REST APIs and HTTP APIs in API Gateway.

**Why it matters:** API Gateway provides centralized API management with built-in security features.

**Pass condition:** APIs exist in API Gateway

**Note:** Informational - APIs may use other patterns (ALB, direct Lambda)

**Remediation:** Consider API Gateway for centralized API management

---

### Check 6.6-B: API Gateway Authorization

| Field | Value |
|-------|-------|
| Severity | HIGH |
| AWS API | `apigateway get-method` |
| Requirement | "OAuth 2.0, OpenID Connect authentication" |

**What it checks:** Verifies API Gateway methods have authorization configured (not NONE).

**Why it matters:** APIs without authorization allow unauthenticated access, violating ZT principles.

**Pass condition:** All APIs have authorization (IAM, Cognito, Lambda, or API Key)

**Fail condition:** APIs with authorizationType=NONE

**Remediation:** Configure IAM authorization, Cognito User Pools, or Lambda authorizers

---

### Check 6.6-C: API Gateway Logging

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `apigateway get-stage` |
| Requirement | "Logging, monitoring, and IR" |

**What it checks:** Verifies API Gateway stages have CloudWatch logging enabled.

**Why it matters:** API logs are essential for security monitoring, troubleshooting, and incident response.

**Pass condition:** Logging enabled on API stages

**Fail condition:** APIs without logging

**Remediation:** Enable CloudWatch logging in API Gateway stage settings

---

### Check 6.6-D: API Gateway WAF Integration

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `wafv2 get-web-acl-for-resource` |
| Requirement | "Security measures" |

**What it checks:** Verifies WAF WebACL is associated with API Gateway stages.

**Why it matters:** WAF protects APIs from common web attacks (SQL injection, XSS, etc.).

**Pass condition:** WAF integrated with APIs

**Note:** Recommended for public-facing APIs

**Remediation:** Create WAF WebACL and associate with API Gateway stage

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| API schema validation | Requires OpenAPI spec review |
| API versioning standards | Requires documentation review |
| Non-compliant API catalog | Requires manual inventory |
| API deprecation process | Requires process documentation |

## External Tools for Full Compliance

To fully implement Activity 6.6.2:
- **API Management** - Kong, Apigee for advanced API governance
- **API Documentation** - Swagger/OpenAPI for schema standardization
- **API Testing** - Postman, SoapUI for compliance testing

## How AWS Provides API Standardization

| ZIG Concept | AWS Equivalent |
|-------------|----------------|
| API Standard | API Gateway with OpenAPI import |
| Authentication | IAM, Cognito, Lambda Authorizers |
| Logging | CloudWatch Logs |
| Security | WAF, Resource Policies |
| Rate Limiting | Usage Plans, Throttling |

## API Gateway Authorization Types

| Type | Use Case |
|------|----------|
| IAM | AWS service-to-service |
| Cognito | User authentication |
| Lambda | Custom authorization logic |
| API Key | Simple client identification |
