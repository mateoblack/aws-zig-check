# Activity 7.1.2 - Log Parsing

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 7.1.2 requires:
- Identify and prioritize log and flow sources
- Establish standardized log format at Enterprise level
- Migrate existing solutions to standard format
- Filter and forward applicable log events to SIEM

## Predecessor/Successor Relationships

```
None → 7.1.2 (Log Parsing) → 7.2.4, 7.3.1
```

**Predecessor:** None
**Successor:** Activity 7.2.4 - Asset ID and Alert Correlation, Activity 7.3.1 - Implement Analytics Tools

## AWS Limitations

AWS provides comprehensive logging but:
- No single "SIEM" service (use CloudWatch, Security Lake, or third-party)
- Log formats vary by service
- Multi-region logging requires explicit configuration

## Checks Implemented

---

### Check 7.1-A: CloudTrail Multi-Region Trail

| Field | Value |
|-------|-------|
| Severity | HIGH/BLOCKER |
| AWS API | `cloudtrail describe-trails` |
| Requirement | "Collect logs from all sources" |

**What it checks:** Verifies CloudTrail is configured as multi-region or organization trail.

**Why it matters:** Single-region trails only capture events in one region. Attackers can operate in other regions undetected. Multi-region trails capture ALL AWS API activity across ALL regions.

**Pass condition:** Multi-region or organization trail exists

**Fail condition:** Only single-region trails or no trails

**Remediation:** 
- Create a new trail with "Apply trail to all regions" enabled
- Or use AWS Organizations trail for multi-account coverage

---

### Check 7.1-B: CloudTrail Log File Validation

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `cloudtrail describe-trails` |
| Requirement | "Ensure log integrity" |

**What it checks:** Verifies log file validation is enabled on CloudTrail trails.

**Why it matters:** Log validation creates digest files that detect if logs have been tampered with or deleted.

**Pass condition:** Log file validation enabled

**Fail condition:** Validation disabled

**Remediation:** Enable log file validation in CloudTrail trail settings

---

### Check 7.1-C: CloudTrail Data Events

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `cloudtrail get-event-selectors` |
| Requirement | "Log data access" |

**What it checks:** Verifies CloudTrail is configured to log data events (S3, Lambda).

**Why it matters:** Management events show who did what to AWS resources. Data events show who accessed the data itself (S3 object reads/writes).

**Pass condition:** Data events configured

**Note:** Optional - can be expensive for high-volume buckets

**Remediation:** Enable data events for critical S3 buckets

---

### Check 7.1-D: CloudWatch Log Groups

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `logs describe-log-groups` |
| Requirement | "Centralized logging" |

**What it checks:** Verifies CloudWatch Log Groups exist for log aggregation.

**Why it matters:** CloudWatch Logs provides centralized log storage, search, and alerting capabilities.

**Pass condition:** Log groups exist

**Fail condition:** No log groups

**Remediation:** Configure applications and services to send logs to CloudWatch

---

### Check 7.1-E: Log Retention Policies

| Field | Value |
|-------|-------|
| Severity | LOW |
| AWS API | `logs describe-log-groups` |
| Requirement | "Log retention policy" |

**What it checks:** Verifies log groups have retention policies configured.

**Why it matters:** Without retention policies, logs are kept forever (cost) or may be deleted unexpectedly.

**Pass condition:** Most log groups have retention policies

**Fail condition:** >50% of log groups have no retention

**Remediation:** Set appropriate retention periods based on compliance requirements

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| Standardized log format | Requires documentation review |
| Log source prioritization | Requires manual assessment |
| SIEM integration | Environment-specific |

## External Tools for Full Compliance

To fully implement Activity 7.1.2:
- **SIEM** - Splunk, Elastic, AWS Security Lake
- **Log Aggregation** - Fluentd, Logstash
- **Log Format** - Common Event Format (CEF), JSON

## How AWS Provides Log Parsing

| ZIG Concept | AWS Equivalent |
|-------------|----------------|
| Log Collection | CloudTrail, CloudWatch Logs, VPC Flow Logs |
| Log Standardization | CloudWatch Logs Insights, Athena |
| SIEM | Security Lake, OpenSearch, third-party |
| Log Integrity | CloudTrail log validation |
