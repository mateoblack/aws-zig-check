# Activity 1.8.1 - Single Authentication

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 1.8.1 requires:
- Authentication implemented at least once per session using CAC or DoW-approved MFA
- Leverage approved IdP from Activity 1.3.1
- All Users/PEs/NPEs included in authentication policy
- Session timeout and termination settings configured per Enterprise policy
- No application/service-based identities and groups (use centralized IdP)

## Relationship to Other Activities

Activity 1.8.1 (Single Authentication) builds on Activity 1.3.1 (MFA/IdP) and is a predecessor to Activity 1.8 (Continuous Authentication). The checks for 1.8.1 validate that the foundational authentication infrastructure from 1.3.1 is in place and properly configured for session-based authentication.

**Dependency Chain:**
```
1.3.1 (MFA/IdP) → 1.8.1 (Single Auth) → 1.8 (Continuous Auth)
```

## Checks Implemented

Activity 1.8.1 validates that prerequisites from 1.3.1 are met and adds session-specific checks.

---

### Check 1.8.1-A: IdP Configured for Session Authentication

| Field | Value |
|-------|-------|
| Severity | HIGH |
| Validates | 1.3.1-C (SSO) or 1.3.1-F (External IdP) |
| Requirement | "Leverage the approved IdP and MFA from Activity 1.3.1" |

**What it checks:** Confirms that either IAM Identity Center (SSO) or external IdP federation (SAML/OIDC) is configured to handle session authentication.

**Why it matters:** ZIG 1.8.1 explicitly requires leveraging the IdP from 1.3.1. Without a centralized IdP, session-based authentication cannot be enforced consistently across resources.

**Pass condition:** IAM Identity Center is configured OR external SAML/OIDC providers exist

**Fail condition:** No centralized IdP configured

**References check:** 1.3.1-C, 1.3.1-F

---

### Check 1.8.1-B: MFA Enforced at Session Initiation

| Field | Value |
|-------|-------|
| Severity | HIGH |
| Validates | 1.3.1-A (Root MFA), 1.3.1-B (User MFA), 1.3.1-E (MFA SCP) |
| Requirement | "Authenticate with MFA at least once per session" |

**What it checks:** Confirms MFA is required for all authentication events:
- Root account has MFA
- All IAM users with console access have MFA
- MFA enforcement SCP exists (if using Organizations)

**Why it matters:** The core requirement of 1.8.1 is "authentication at least once per session" using MFA. This check validates MFA is universally enforced.

**Pass condition:** All MFA checks from 1.3.1 pass

**Fail condition:** Any MFA gap exists

**References checks:** 1.3.1-A, 1.3.1-B, 1.3.1-E

---

### Check 1.8.1-C: Session Timeout Configuration

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `iam list-roles`, `iam get-role` |
| Requirement | "Configure session timeout and termination settings" |

**What it checks:** Validates IAM role session durations are configured with reasonable timeouts (not maximum 12 hours).

**Why it matters:** ZIG requires "session timeout and termination settings based on inactivity thresholds." Long session durations (12 hours) effectively bypass re-authentication requirements.

**Pass condition:** IAM roles have MaxSessionDuration ≤ 4 hours (14400 seconds)

**Fail condition:** Roles allow sessions > 4 hours without documented justification

**Note:** This check is shared with Activity 1.8 (Continuous Authentication) check 1.8-A.

---

### Check 1.8.1-D: No Local Service Accounts (Use IdP)

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| Validates | 1.3.1-D (Local users when SSO exists) |
| Requirement | "Components do not use application/service-based identities and groups" |

**What it checks:** When IAM Identity Center is configured, validates that local IAM users with console access have been retired.

**Why it matters:** ZIG 1.8.1 states "Components do not use application/service-based identities and groups." Local IAM users bypass the centralized IdP and cannot be managed through enterprise authentication policies.

**Pass condition:** No local IAM users with console access when SSO is configured

**Fail condition:** Local IAM users exist alongside SSO (except documented break-glass accounts)

**References check:** 1.3.1-D

---

### Check 1.8.1-E: Authentication Logging Enabled

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `cloudtrail describe-trails`, `cloudtrail get-trail-status` |
| Requirement | "Verify and validate that Users/PEs are authenticated" |

**What it checks:** Confirms CloudTrail is enabled to log authentication events (ConsoleLogin, AssumeRole, etc.).

**Why it matters:** ZIG requires ability to "demonstrate that a User/PE is required to authenticate when accessing a Component resource." Without CloudTrail, authentication events cannot be audited or verified.

**Pass condition:** Multi-region CloudTrail trail is active

**Fail condition:** No active CloudTrail logging

**Note:** This check is shared with Activity 1.8 (Continuous Authentication) check 1.8-C.

## Summary: Check Cross-References

| 1.8.1 Check | References | New Logic? |
|-------------|------------|------------|
| 1.8.1-A (IdP Configured) | 1.3.1-C, 1.3.1-F | No - validates prior results |
| 1.8.1-B (MFA Enforced) | 1.3.1-A, 1.3.1-B, 1.3.1-E | No - validates prior results |
| 1.8.1-C (Session Timeout) | 1.8-A | No - same check |
| 1.8.1-D (No Local Accounts) | 1.3.1-D | No - validates prior results |
| 1.8.1-E (Auth Logging) | 1.8-C | No - same check |

**Implementation Note:** The `check_1_8_1_single_auth()` function reports the status of these prerequisite checks rather than re-running them. This provides visibility that Activity 1.8.1 was evaluated while avoiding duplicate API calls.

---

## Checks NOT Implemented

| Requirement | Reason |
|-------------|--------|
| CAC/PIV authentication verification | External IdP configuration - outside AWS API |
| Inactivity timeout enforcement | Application-level setting - varies by service |
| NPE (Non-Person Entity) authentication | Requires service account inventory - manual review |
| Policy/procedure documentation | Process artifact - not API queryable |
