# Activity 7.2.1 - Threat Alerting Part 1

## ZIG Requirement Summary

Per NSA ZIG Phase One, Activity 7.2.1 requires:
- Develop SIEM rules and alerts for common threat events
- Create rules for component-derived threat correlation
- Develop rules for asset ID-based responses
- Prepare automated response actions

## Predecessor/Successor Relationships

```
None → 7.2.1 (Threat Alerting Part 1) → 2.7.2, 7.2.2
```

**Predecessor:** None
**Successor:** Activity 2.7.2 - Implement XDR, Activity 7.2.2 - Threat Alerting Part 2

## AWS Limitations

AWS provides alerting through multiple services:
- CloudWatch Alarms for metric-based alerts
- CloudWatch Metric Filters for log-based alerts
- GuardDuty for threat detection (covered in Pillar 6)
- Security Hub for aggregated findings (covered in Pillar 6)

## Checks Implemented

---

### Check 7.2-A: CloudTrail Insights

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `cloudtrail get-insight-selectors` |
| Requirement | "Anomaly detection" |

**What it checks:** Verifies CloudTrail Insights is enabled for anomalous API activity detection.

**Why it matters:** Insights uses ML to detect unusual API call patterns that may indicate compromise.

**Pass condition:** Insights enabled on at least one trail

**Note:** Optional - additional cost

**Remediation:** Enable Insights in CloudTrail trail settings

---

### Check 7.2-B: CloudWatch Metric Filters

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `logs describe-metric-filters` |
| Requirement | "Rules for threat correlation" |

**What it checks:** Verifies CloudWatch metric filters exist for log-based alerting.

**Why it matters:** Metric filters extract metrics from logs (e.g., count of unauthorized API calls) that can trigger alarms.

**Pass condition:** Metric filters configured

**Fail condition:** No metric filters

**Remediation:** Create metric filters for security events in CloudTrail logs

---

### Check 7.2-C: SNS Topics for Alerting

| Field | Value |
|-------|-------|
| Severity | INFO |
| AWS API | `sns list-topics` |
| Requirement | "Alert notification" |

**What it checks:** Verifies SNS topics exist for alert delivery.

**Why it matters:** SNS topics deliver alerts to email, SMS, Lambda, or other endpoints.

**Pass condition:** SNS topics exist

**Note:** Informational - topics may be used for non-security purposes

---

### Check 7.2-D: CloudWatch Alarms

| Field | Value |
|-------|-------|
| Severity | MEDIUM |
| AWS API | `cloudwatch describe-alarms` |
| Requirement | "Threat alerting" |

**What it checks:** Verifies CloudWatch alarms are configured.

**Why it matters:** Alarms trigger notifications when metrics exceed thresholds.

**Pass condition:** Alarms exist

**Fail condition:** No alarms configured

**Remediation:** Create alarms for security metrics

## Recommended Metric Filters

For ZIG-compliant threat alerting, create these CloudWatch metric filters on CloudTrail logs:

| Alert | Filter Pattern |
|-------|---------------|
| Root Login | `{ $.userIdentity.type = "Root" && $.userIdentity.invokedBy NOT EXISTS && $.eventType != "AwsServiceEvent" }` |
| Console Login Failures | `{ $.eventName = "ConsoleLogin" && $.errorMessage = "Failed authentication" }` |
| Unauthorized API Calls | `{ $.errorCode = "*UnauthorizedAccess*" || $.errorCode = "AccessDenied*" }` |
| IAM Policy Changes | `{ $.eventName = "DeleteGroupPolicy" || $.eventName = "DeleteRolePolicy" || $.eventName = "PutGroupPolicy" }` |
| CloudTrail Changes | `{ $.eventName = "StopLogging" || $.eventName = "DeleteTrail" || $.eventName = "UpdateTrail" }` |
| Security Group Changes | `{ $.eventName = "AuthorizeSecurityGroupIngress" || $.eventName = "RevokeSecurityGroupIngress" }` |
| Network ACL Changes | `{ $.eventName = "CreateNetworkAcl" || $.eventName = "DeleteNetworkAcl" }` |
| S3 Bucket Policy Changes | `{ $.eventName = "PutBucketPolicy" || $.eventName = "DeleteBucketPolicy" }` |
